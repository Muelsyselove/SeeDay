# 部署要求

本文档描述部署 SeeDay 所需的硬件和软件要求。

## 服务器配置

### 最低配置

| 资源 | 要求 | 说明 |
|------|------|------|
| CPU | 2 核 | 处理 HTTP 请求和数据库操作 |
| 内存 | 4 GB | 运行 Bun 后端和前端静态文件 |
| 存储 | 20 GB | 系统 + 应用程序 + 数据库 |
| 网络 | 100 Mbps | Agent 数据上报和前端访问 |

### 推荐配置

| 资源 | 要求 | 说明 |
|------|------|------|
| CPU | 2-4 核 | 更好的并发处理能力 |
| 内存 | 4-8 GB | 更充裕的内存空间 |
| 存储 | **60 GB** | 充足的存储空间，支持长期数据保留 |
| 网络 | 1 Gbps | 更快的数据传输 |

### 存储说明

SeeDay 的存储主要用于：

| 用途 | 预估空间 | 说明 |
|------|---------|------|
| 操作系统 | 10-15 GB | Linux 系统基础 |
| 应用程序 | 1-2 GB | Docker 镜像/Node.js/Python |
| 数据库 | 动态增长 | 取决于活动数据量和保留时间 |
| 日志文件 | 1-5 GB | 应用日志和访问日志 |
| 备份 | 按需 | 数据库备份文件 |
| **推荐总计** | **60 GB** | 留有足够的缓冲空间 |

> **建议**：配置 60 GB 存储空间可以支持约 10-20 台设备、每天 24 小时活动追踪、保留 30 天数据的场景。

## 网络要求

### 端口

| 端口 | 协议 | 用途 | 是否必须开放 |
|------|------|------|-------------|
| 3000 | TCP | 后端 API 服务 | 否（内网） |
| 3001 | TCP | 前端服务 | 否（内网） |
| 80 | TCP | HTTP 入口 | 是（如需 HTTP） |
| 443 | TCP | HTTPS 入口 | 是（推荐） |

### 防火墙配置

确保以下端口可访问：

```bash
# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 如果使用 Docker，确保 Docker 网络可以通信
sudo ufw allow in on docker0
```

### DNS 配置

建议配置域名解析：

```
see.yourdomain.com  →  服务器 IP
```

## 安全建议

1. **使用 HTTPS**：始终使用 HTTPS 加密传输数据
2. **强 Token**：使用足够长的随机 Token（建议 32+ 字符）
3. **定期更新**：保持系统和依赖软件最新
4. **防火墙**：仅开放必要的端口
5. **备份**：定期备份数据库文件
6. **监控**：监控服务器资源使用情况
