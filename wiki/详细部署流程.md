## 1. 准备阶段

在开始部署之前，请确保你的环境满足要求。

### 1.1 检查系统环境

**前置条件**：
- 拥有 Linux 服务器（推荐 Ubuntu 20.04+ 或 Debian 11+）
- 具有 root 或 sudo 权限
- 服务器内存至少 4GB，存储至少 20GB

**执行命令**：
```bash
# 检查操作系统版本
cat /etc/os-release

# 检查内存
free -h

# 检查存储空间
df -h
```

**预期输出**：
- `cat /etc/os-release`: 显示 Ubuntu 或 Debian 版本信息
- `free -h`: 显示 Mem: 至少 4Gi
- `df -h`: 显示可用空间至少 20G

### 1.2 安装基础工具

如果服务器未安装 `git` 或 `curl`，请先安装：

```bash
# Ubuntu/Debian
sudo apt update && sudo apt install -y git curl

# CentOS/Rocky
sudo yum update && sudo yum install -y git curl
```

## 2. 服务器部署（推荐 Docker）

这是最简单快捷的方式，适合大多数用户。

### 2.1 安装 Docker

**前置条件**：
- 干净的 Linux 服务器

**执行命令**：
```bash
# 一键安装 Docker
curl -fsSL https://get.docker.com | bash

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
sudo docker --version
```

**预期输出**：
```
Docker version 24.x.x, build xxxxx
```

### 2.2 生成密钥

**前置条件**：
- Docker 已安装

**执行命令**：
```bash
# 生成 HASH_SECRET（哈希密钥）
HASH_SECRET=$(openssl rand -hex 32)
echo "HASH_SECRET=$HASH_SECRET"

# 生成 DEVICE_TOKEN（设备 Token，格式：token:id:name:platform）
TOKEN=$(openssl rand -hex 16)
DEVICE_TOKEN="${TOKEN}:my-pc:My PC:windows"
echo "DEVICE_TOKEN=$DEVICE_TOKEN"
```

**预期输出**：
- 显示两行随机生成的密钥字符串。**请妥善保存这些值！**

### 2.3 创建配置文件

**前置条件**：
- 已生成密钥

**执行命令**：
```bash
# 创建项目目录
mkdir -p ~/seeday && cd ~/seeday

# 创建 .env 文件（替换下方变量为你实际生成的值）
cat > .env <<EOF
HASH_SECRET=你的 HASH_SECRET 值
DEVICE_TOKEN_1=你的 DEVICE_TOKEN 值
# AI 总结配置（可选）
AI_API_URL=https://api.openai.com/v1/chat/completions
AI_API_KEY=sk-你的 API Key
AI_MODEL=gpt-4o-mini
EOF

# 创建 docker-compose.yml
cat > docker-compose.yml <<'EOF'
services:
  dashboard:
    image: ghcr.io/muelsyselove/seeday:latest
    container_name: seeday
    restart: unless-stopped
    network_mode: host
    environment:
      - PORT=3000
      - STATIC_DIR=/app/public
      - DB_PATH=/data/live-dashboard.db
      - DEVICE_TOKEN_1=${DEVICE_TOKEN_1}
      - HASH_SECRET=${HASH_SECRET}
      - AI_API_URL=${AI_API_URL:-}
      - AI_API_KEY=${AI_API_KEY:-}
      - AI_MODEL=${AI_MODEL:-gpt-4o-mini}
    volumes:
      - seeday_data:/data
    mem_limit: 256m
    cpus: 1.0

volumes:
  seeday_data:
EOF
```

### 2.4 启动服务

**执行命令**：
```bash
# 启动服务
sudo docker compose up -d

# 查看日志
sudo docker compose logs -f
```

**预期输出**：
- `docker compose up -d`: 显示 `Container seeday Started`
- `logs -f`: 显示服务启动成功信息，无报错

**访问验证**：
在浏览器访问 `http://你的服务器IP:3000`，应能看到 SeeDay 仪表盘界面。

## 3. 服务器部署（手动方式）

如果你不想使用 Docker，可以选择手动部署。

### 3.1 安装 Bun 运行时

**执行命令**：
```bash
# 安装 Bun
curl -fsSL https://bun.sh/install | bash

# 使环境变量生效
source ~/.bashrc

# 验证
bun --version
```

### 3.2 克隆并配置项目

**执行命令**：
```bash
# 克隆项目
git clone https://github.com/Muelsyselove/SeeDay.git ~/seeday
cd ~/seeday

# 配置后端
cd packages/backend
bun install

# 创建 .env
cp .env.example .env
nano .env  # 编辑配置，填入 HASH_SECRET 和 DEVICE_TOKEN_1
```

### 3.3 构建并启动

**执行命令**：
```bash
# 构建前端
cd ~/seeday/packages/frontend
bun install
bun run build

# 复制前端产物到后端 public 目录
cp -r out/* ../backend/public/

# 启动服务
cd ../backend
bun run src/index.ts
```

## 4. Agent 部署（Windows 客户端）

Agent 运行在你的电脑上，用于追踪应用使用情况并上报给服务器。

### 4.1 下载 Agent

**方式一：从 Release 下载（推荐）**
1. 访问 [SeeDay Releases](https://github.com/Muelsyselove/SeeDay/releases)
2. 下载最新的 `Windows-Agent.zip` 或 `.exe` 文件
3. 解压到任意目录（如 `C:\SeeDayAgent`）

**方式二：源码编译**
1. 确保安装了 Python 3.8+
2. 克隆仓库：`git clone https://github.com/Muelsyselove/SeeDay.git`
3. 进入目录：`cd SeeDay/agents/windows`
4. 安装依赖：`pip install -r requirements.txt`
5. 打包：运行 `build.bat` 生成可执行文件

### 4.2 配置 Agent

**前置条件**：
- 已下载/编译 Agent
- 已从服务器 `.env` 获取 `DEVICE_TOKEN_1` 的值
- 已知服务器 IP 地址

**执行步骤**：
1. 在 Agent 目录创建 `config.json`
2. 编辑内容如下：

```json
{
  "server_url": "http://你的服务器IP:3000",
  "token": "从服务器 DEVICE_TOKEN_1 中提取的 token 部分（冒号前）",
  "interval_seconds": 5,
  "heartbeat_seconds": 60,
  "idle_threshold_seconds": 300
}
```

**配置项详解**：
| 字段 | 说明 | 示例 |
|------|------|------|
| `server_url` | 后端服务器地址 | `http://123.123.123.123:3000` |
| `token` | 认证 Token（冒号前的部分） | `abc123def456...` |
| `interval_seconds` | 数据上报间隔（秒） | `5` |
| `heartbeat_seconds` | 心跳间隔（秒） | `60` |
| `idle_threshold_seconds` | 空闲判定阈值（秒） | `300` |

### 4.3 运行 Agent

**执行步骤**：
1. 双击 `agent.exe`（或运行 `python agent.py`）
2. 系统托盘会出现 SeeDay 图标
3. 右键图标可以打开设置或退出

**预期效果**：
- 托盘图标显示（在线为绿色，离线为灰色）
- 图标提示显示当前运行的前台/后台应用

## 5. 验证与排查

### 5.1 验证连通性

**检查项 1：Agent 日志**
- 查看 Agent 目录下的 `agent_YYYY-MM-DD.log`
- 搜索关键字 `Reported`，确认数据已发送

**检查项 2：服务器日志**
```bash
sudo docker compose logs seeday | grep "Report"
```
- 应看到类似 `Received report from device...` 的日志

**检查项 3：Web 界面**
- 刷新仪表盘页面
- 查看是否有设备卡片显示，且状态为"在线"

### 5.2 常见问题排查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 401 Unauthorized | Token 不匹配 | 检查 `.env` 的 `DEVICE_TOKEN_1` 和 `config.json` 的 `token` 是否一致 |
| 404 Not Found | server_url 错误 | 检查 URL 是否包含 `/` 结尾或端口是否正确 |
| Connection Refused | 端口未开放/防火墙 | 检查 `ufw status`，开放 3000 端口 |
| Agent 图标灰色 | 网络不通 | 检查服务器 IP 是否可达，使用 `ping` 测试 |
| 413 Payload Too Large | 数据过大 | 检查是否有大量后台应用，调整 `interval_seconds` |

### 5.3 防火墙配置示例

```bash
# 开放 3000 端口
sudo ufw allow 3000/tcp

# 如果使用 Nginx，开放 80/443
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```
