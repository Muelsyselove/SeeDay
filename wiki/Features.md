# 功能介绍

SeeDay 提供了一套完整的个人时间追踪和可视化解决方案，以下是核心功能的详细介绍。

## 实时活动追踪

### 功能描述

SeeDay 通过安装在设备上的轻量级 Agent 程序，实时监测并记录用户当前正在使用的应用程序。

### 核心能力

- **前台应用检测**：自动识别当前处于前台的应用程序
- **应用名称映射**：将应用路径映射为人类可读的应用名称
- **实时更新**：Agent 定期向服务器报告当前活动状态
- **多设备支持**：支持同时追踪多个设备的活动数据

### Agent 平台支持

| 平台 | 状态 | 技术栈 |
|------|------|--------|
| Windows | ✅ 已支持 | Python + Win32 API + pystray |
| macOS | 🔨 开发中 | Python + AppleScript + pystray |
| iOS | 🔨 开发中 | 计划中 |
| Android | 🔨 开发中 | 计划中 |

### Windows Agent 功能

- 系统托盘常驻运行
- AFK（离开键盘）检测
- 看视频/听歌时自动豁免追踪
- 音乐播放器检测（Spotify、QQ音乐、网易云等）
- 三级隐私系统支持

## 可视化仪表盘

### 功能描述

精美的 Web 界面，实时展示各设备的活动状态和使用统计。

### 界面元素

- **当前状态展示**：显示所有设备的实时活动状态
- **活动时间线**：按时间顺序展示每日活动记录
- **应用使用统计**：Top 应用使用时长图表
- **设备卡片**：每个设备一张卡片，显示在线状态和活动信息

### 主题系统

SeeDay 支持多种主题风格，可自由切换：

| 主题 | 风格 | 说明 |
|------|------|------|
| 默认主题 | 现代简洁 | 干净的现代设计风格 |
| 水墨主题 | 中国传统 | 水墨国风，宣纸质感，朱砂红点缀 |
| 更多主题 | 持续添加 | 社区贡献主题 |

主题切换方式：
- 在仪表盘设置中切换
- 使用快捷键快速切换

### 响应式设计

- 桌面端：完整的双栏/多栏布局
- 平板端：自适应单栏布局
- 手机端：优化的移动端体验

## AI 每日总结

### 功能描述

每晚自动汇总当日活动数据，调用 AI API 生成简洁的中文活动总结。

### 工作流程

1. 每晚 21:00 定时触发
2. 汇总当日所有活动数据
3. 调用 AI API 生成 50-80 字中文总结
4. 存储到数据库并在仪表盘展示

### 支持的 AI 提供商

| 提供商 | API 端点 | 推荐模型 |
|--------|---------|---------|
| OpenAI | `https://api.openai.com/v1/chat/completions` | gpt-4o-mini |
| Claude | 代理地址 | claude-3-haiku-20240307 |
| DeepSeek | `https://api.deepseek.com/chat/completions` | deepseek-chat |
| 本地 Ollama | `http://localhost:11434/v1/chat/completions` | qwen2.5:7b |

### 配置方式

在 `.env` 文件中设置：

```
AI_API_URL=https://api.openai.com/v1/chat/completions
AI_API_KEY=sk-...
AI_MODEL=gpt-4o-mini
```

> 不配置 AI 时，AI 总结功能静默跳过，不影响其他功能。

## 隐私说明

### 当前隐私处理机制

本项目提供了以下基础的数据处理机制，用于在仪表盘中隐藏部分敏感内容：

| 机制 | 说明 | 局限性 |
|------|------|--------|
| 应用分级 | 根据应用名称将内容分为 SHOW / BROWSER / HIDE 三级 | 基于应用名称的简单匹配，可能被绕过 |
| 敏感词过滤 | 浏览器标题中包含敏感关键词时自动隐藏 | 关键词列表可能不完整，无法覆盖所有场景 |
| NSFW 过滤 | 基于预定义列表过滤不适当内容 | 依赖人工维护的列表，覆盖有限 |
| 浏览器后缀剥离 | 从浏览器标签页中移除浏览器名称 | 仅针对已知浏览器 |

### 隐私风险提示

> **重要声明**：本项目并非专业的隐私保护工具，**不保证数据的完全安全**。使用前请了解以下可能存在的隐私问题：

1. **窗口标题可能泄露敏感信息**：Agent 会捕获前台窗口的标题文字，包括但不限于：
   - 聊天消息预览（如微信、QQ、Discord 等）
   - 邮件主题和摘要
   - 文档和文件名称
   - 浏览的网页标题
   - 登录和认证页面

2. **应用分级可能被绕过**：隐私分级基于应用名称匹配，新应用或更名后的应用可能不被正确分类。

3. **AI 总结涉及数据外传**：如果启用了 AI 每日总结功能，当日活动数据会被发送到第三方 AI 服务提供商（如 OpenAI、Claude 等），这些数据不在本地控制范围内。

4. **网络传输安全依赖部署配置**：Agent 到后端服务器的数据传输依赖于你的部署配置。如果使用 HTTP 而非 HTTPS，数据在传输过程中可能被窃听。

5. **日志可能包含敏感信息**：应用日志中可能记录了活动数据，如果未正确配置日志清理策略，可能长期保留敏感信息。

6. **数据库未加密**：活动数据以明文形式存储在 SQLite 数据库中，如果服务器被入侵，所有数据可能被直接读取。

7. **多设备数据汇总**：如果你允许多人访问同一仪表盘，所有设备的活动数据都会被展示。

### 建议的最佳实践

- 始终使用 HTTPS 进行数据传输
- 定期备份并清理旧数据
- 不要将仪表盘暴露在公开网络
- 谨慎使用 AI 总结功能（涉及数据外传）
- 定期审查应用分级配置
- 限制数据库文件的访问权限
- 了解并接受上述风险后再使用

## 多设备支持

### 功能描述

支持同时管理多个设备的活动数据，在统一的仪表盘中查看所有设备状态。

### 设备管理

- 每个设备独立的 Token 认证
- 设备名称自定义
- 设备类型标识（Windows/macOS/其他）
- 在线/离线状态实时显示

### 离线处理

- 设备离线时显示友好的离线状态
- 自动检测长时间离线
- 夜间模式自动切换（设备全离线时）

## 数据统计

### 活动聚合

- 相同应用的连续活动自动合并
- 按应用分组的时间线展示
- "此刻"实时汇总置顶显示

### 数据清理

- 自动清理过期数据
- AI 总结 7 天自动清理
- 可配置的保留策略
