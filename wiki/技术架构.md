## 数据流转

### 活动数据上报流程

```
1. Agent 检测前台应用（Win32 API GetForegroundWindow）
       │
       ▼
2. 通过 psutil 获取进程名称
       │
       ▼
3. 黑名单过滤后台系统进程
       │
       ▼
4. 媒体检测（音乐/视频进程识别）
       │
       ▼
5. 隐私模式判断
       │
       ▼
6. 后台应用枚举（EnumWindows + psutil）
       │
       ▼
7. HTTP POST 到后端 /api/report
       │
       ▼
8. 后端验证 Token（HMAC 哈希认证）
       │
       ▼
9. 写入 SQLite 数据库
       │
       ▼
10. 前端轮询 /api/timeline 获取数据
       │
       ▼
11. 渲染到仪表盘
```

### API 接口设计

#### 认证方式

所有 Agent 请求使用 Token 认证：
```
Authorization: Bearer <DEVICE_TOKEN>
```

Token 格式：`token:device_id:device_name:platform`

#### 核心端点

| 端点 | 方法 | 说明 | 认证 |
|------|------|------|------|
| `/api/report` | POST | 上报活动数据 | 需要 |
| `/api/current` | GET | 获取当前活动 | 可选 |
| `/api/timeline` | GET | 获取时间线数据 | 可选 |
| `/api/daily-summary` | GET | 获取每日 AI 总结 | 可选 |
| `/api/health` | GET | 健康检查 | 无需 |

#### 数据模型

**活动记录表（activities）**

存储设备上报的活动数据，包含应用名称、窗口标题、时间戳等。

**每日总结表（daily_summaries）**

存储 AI 生成的每日活动总结。

**设备表（devices）**

存储设备认证信息和元数据。

## 核心服务

### 认证服务（Token Verification）

验证 Agent 上报数据的身份认证。使用 HMAC 哈希保护窗口标题数据。

### 每日总结生成服务（Daily Summary）

定时任务（每晚 21:00），汇总活动数据并调用 AI API 生成中文总结。

### 数据清理服务

定期清理过期数据，保持数据库整洁。AI 总结默认保留 7 天。

## Windows Agent 高级功能

### AFK 检测

通过 `GetLastInputInfo` API 检测用户是否离开键盘/鼠标。超过阈值后切换到心跳模式。

### 媒体检测

扫描所有窗口标题，识别已知的音乐播放器（Spotify、QQ音乐、网易云等），提取歌曲信息。

### 隐私模式

托盘菜单可切换隐私模式。开启后，上报数据替换为通用描述，仪表盘不暴露具体应用。

### 背景应用枚举

通过 `EnumWindows` + `psutil.process_iter` 双重扫描，获取前台和后台运行中的应用列表。

### URL 安全验证

Agent 内置 URL 安全校验：
- HTTPS 始终允许
- HTTP 仅允许连接私有网络（192.168.x.x, 10.x.x.x, localhost）
- 禁止 HTTP 连接公网 IP
