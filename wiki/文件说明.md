## 1. 服务器部署必需文件

以下文件是部署到服务器（VPS/云服务器）时**必须包含**的：

### 1.1 核心目录结构

```
SeeDay/
├── packages/
│   ├── backend/            # 🔴 必需 - 后端服务
│   │   ├── src/            # 后端源码
│   │   │   ├── index.ts    # 入口文件
│   │   │   ├── db.ts       # 数据库操作
│   │   │   └── ...         # 路由、服务等
│   │   ├── package.json    # 依赖配置
│   │   └── bun.lock*       # 锁定文件
│   └── frontend/           # 🔴 必需 - 前端源码
│       ├── app/            # Next.js 应用
│       ├── src/            # React 组件
│       ├── package.json    # 依赖配置
│       └── out/            # 🟡 构建产物（手动部署时需要）
├── Dockerfile              # 🔴 必需 - Docker 镜像构建
├── docker-compose.yml      # 🔴 必需 - Docker 编排配置
├── .env.example            # 🟡 可选 - 环境变量模板
├── start.sh                # 🟡 可选 - 启动脚本
└── data/                   # 🟡 自动创建 - SQLite 数据库目录
```

### 1.2 Docker 部署时需要的文件

如果你使用 **Docker 部署**，只需要：

| 文件 | 必需 | 说明 |
|------|------|------|
| `Dockerfile` | ✅ | 定义构建过程 |
| `docker-compose.yml` | ✅ | 定义容器运行 |
| `.env` | ✅ | 环境变量配置 |
| `packages/backend/` | ✅ | 后端源码（构建时复制进镜像） |
| `packages/frontend/` | ✅ | 前端源码（构建时复制进镜像） |

Docker 会自动完成构建，你不需要手动处理其他文件。

### 1.3 手动部署时需要的文件

如果你使用 **手动部署**（Bun + Node.js），需要：

| 文件/目录 | 必需 | 说明 |
|-----------|------|------|
| `packages/backend/` | ✅ | 后端源码和依赖 |
| `packages/frontend/out/` | ✅ | 前端构建产物 |
| `packages/backend/.env` | ✅ | 环境变量 |
| `data/` | ✅ | SQLite 数据存储目录 |

## 2. Agent 端文件

以下文件仅需部署到 **客户端电脑**（你的 Windows 电脑），不需要放到服务器：

### 2.1 Windows Agent 文件

```
agents/windows/
├── agent.py              # 🔴 必需 - Agent 主程序
├── config.json           # 🔴 必需 - Agent 配置（需手动创建）
├── requirements.txt      # 🟡 仅开发 - Python 依赖列表
├── build.bat             # 🟡 仅开发 - PyInstaller 打包脚本
├── install-task.bat      # 🟡 仅开发 - 安装计划任务脚本
└── test_tray.py          # 🔵 仅开发 - 测试文件
```

**普通用户只需要**：
- `agent.exe`（预编译版）或 `agent.py`（源码运行）
- `config.json`（配置文件）

**开发者需要**（编译打包）：
- 所有 `.py` 文件
- `requirements.txt`
- `build.bat`

### 2.2 macOS Agent 文件

```
agents/macos/
├── agent.py              # 🔴 必需 - macOS Agent 主程序
├── config.json.example   # 🟡 配置示例
└── requirements.txt      # 🟡 Python 依赖
```

> ⚠️ macOS Agent 尚在开发中，可能不完整。

### 2.3 Android Agent 文件

```
agents/android-agent/
├── app/                  # Android 应用源码
├── gradle/               # Gradle 构建
├── build.gradle.kts      # 构建配置
└── ...                   # Android 项目结构
```

> ⚠️ Android Agent 尚在开发中。

## 3. 克隆仓库后多余/可忽略文件

当你 `git clone` 整个仓库后，以下文件/目录**不需要关注**，可以忽略：

### 3.1 开发工具配置

| 文件/目录 | 说明 |
|-----------|------|
| `.trae/` | Trae IDE 的 spec 和配置 |
| `.trae-cn-server/` | Trae 服务器扩展 |
| `.git/` | Git 版本控制（克隆后自动生成，不需要关注） |

### 3.2 文档和说明

| 文件/目录 | 说明 |
|-----------|------|
| `README.md` | 项目说明文档（阅读即可，无需部署） |
| `CHANGELOG.md` | 更新日志 |
| `LICENSE` | 许可证文件 |
| `updata vision.md` | 更新记录 |
| `wiki/` | Wiki 文档（阅读即可，无需部署） |
| `docs/` | 图片等资源文件 |

### 3.3 开发辅助文件

| 文件/目录 | 说明 |
|-----------|------|
| `merge-activities.js` | 数据合并工具脚本 |
| `merge-all-activities.js` | 全量合并脚本 |
| `dashboard.conf.bak` | 旧版配置备份 |
| `packages/frontend/themes/GUIDE.md` | 主题开发指南 |

### 3.4 构建和缓存文件

| 文件/目录 | 说明 |
|-----------|------|
| `agents/windows/__pycache__/` | Python 编译缓存，可删除 |
| `node_modules/` | Node.js 依赖（`npm install` 或 `bun install` 自动生成） |

### 3.5 环境变量文件

| 文件 | 说明 |
|------|------|
| `.env` | 本地环境变量（**不要提交到 Git**，服务器上单独创建） |
| `.env.example` | 环境变量模板（可作为参考） |

## 4. 项目整体结构

```
SeeDay/
├── agents/                     # Agent 程序（客户端使用）
│   ├── windows/               # Windows Agent ✅ 可用
│   ├── macos/                 # macOS Agent 🔨 开发中
│   └── android-agent/         # Android Agent 🔨 开发中
│
├── packages/                   # 前后端服务（服务器部署使用）
│   ├── backend/               # 后端服务（Bun + TypeScript）
│   └── frontend/              # 前端仪表盘（Next.js + React）
│
├── Dockerfile                  # Docker 镜像构建配置
├── docker-compose.yml          # Docker 编排配置
├── .env.example               # 环境变量模板
├── .gitignore                 # Git 忽略规则
├── start.sh                   # 启动脚本（可选）
├── data/                      # SQLite 数据目录（自动创建）
│
├── README.md                  # 项目说明
├── CHANGELOG.md               # 更新日志
├── LICENSE                    # 许可证
└── wiki/                      # Wiki 文档目录
```

## 快速总结

| 角色 | 需要的文件 |
|------|-----------|
| **服务器部署（Docker）** | `Dockerfile` + `docker-compose.yml` + `.env` |
| **服务器部署（手动）** | `packages/backend/` + `packages/frontend/out/` + `.env` |
| **Windows 客户端** | `agent.exe`（或 `agent.py`）+ `config.json` |
| **开发者** | 整个仓库 + `npm install`/`bun install` |
