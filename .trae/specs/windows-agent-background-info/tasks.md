# Windows Agent 后台信息上报 - 实现计划

## [ ] 任务 1：修改 Windows agent 收集后台应用信息
- **优先级**：P0
- **依赖**：None
- **描述**：
  - 扩展 Windows agent 代码，使其能够收集后台应用信息
  - 实现后台应用的识别逻辑
  - 修改上报逻辑，同时上报前台和后台应用信息
- **验收标准**：AC-1
- **测试要求**：
  - `programmatic` TR-1.1: Windows agent 能够正确识别前台和后台应用
  - `programmatic` TR-1.2: Windows agent 能够同时上报前台和后台应用信息
- **注意**：需要确保后台应用的收集不会显著增加系统资源消耗

## [ ] 任务 2：更新 Windows agent 上报逻辑
- **优先级**：P0
- **依赖**：任务 1
- **描述**：
  - 修改 Reporter 类，支持同时上报多个应用信息
  - 确保前台应用的 `is_foreground` 字段设置为 true，后台应用设置为 false
  - 保持上报频率不变，避免增加服务器负载
- **验收标准**：AC-1, AC-2
- **测试要求**：
  - `programmatic` TR-2.1: 上报数据中 `is_foreground` 字段设置正确
  - `programmatic` TR-2.2: 后台应用信息能够正确发送到服务器
- **注意**：需要考虑上报数据的大小，避免超过服务器的处理能力

## [ ] 任务 3：验证服务器端处理逻辑
- **优先级**：P0
- **依赖**：任务 2
- **描述**：
  - 检查服务器端 `handleReport` 函数是否能够正确处理 `is_foreground` 字段
  - 确保数据库操作正确设置 `is_foreground` 标志
  - 验证后台应用信息能够正确存储
- **验收标准**：AC-2
- **测试要求**：
  - `programmatic` TR-3.1: 服务器能够正确处理包含 `is_foreground: false` 的上报数据
  - `programmatic` TR-3.2: 数据库中的 `ActivityRecord` 和 `DeviceState` 正确设置了 `is_foreground` 字段
- **注意**：服务器端逻辑可能已经支持 `is_foreground` 字段，需要验证其功能

## [ ] 任务 4：更新前端界面显示逻辑
- **优先级**：P1
- **依赖**：任务 3
- **描述**：
  - 修改前端组件，根据 `is_foreground` 字段显示不同的状态文本
  - 前台应用显示“正在查看”，后台应用显示“正在运行”
  - 确保状态显示在设备卡片和时间线组件中
- **验收标准**：AC-3
- **测试要求**：
  - `human-judgment` TR-4.1: 前台应用显示“正在查看”
  - `human-judgment` TR-4.2: 后台应用显示“正在运行”
  - `human-judgment` TR-4.3: 状态显示在所有相关组件中
- **注意**：需要确保前端界面的响应速度不受影响

## [ ] 任务 5：测试和验证
- **优先级**：P1
- **依赖**：任务 4
- **描述**：
  - 在测试环境中运行 Windows agent
  - 验证前台和后台应用信息的上报
  - 检查服务器端数据处理
  - 测试前端界面显示
- **验收标准**：AC-1, AC-2, AC-3
- **测试要求**：
  - `programmatic` TR-5.1: 完整的端到端测试，验证数据流程
  - `human-judgment` TR-5.2: 手动测试前端界面显示
- **注意**：需要模拟不同的应用场景，确保功能的稳定性
