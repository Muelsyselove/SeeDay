# Live Dashboard 记录逻辑重构 - 验证清单

- [ ] 客户端 agent 能够正确提交新格式的数据，使用分号分隔的格式提交多个应用信息
- [ ] 服务器能够正确接收和解析新格式的数据，提取多个应用的信息
- [ ] 服务器能够基于前后台状态判断应用是否持续运行，当应用在一次提交中不在前台或后台运行时视为应用结束
- [ ] 服务器能够正确处理跨天的情况，在第一天结束所有条目，在第二天创建新条目
- [ ] 服务器能够正确处理媒体应用的特殊规则，包括音乐和视频应用
- [ ] 所有功能测试通过，系统性能和稳定性良好
- [ ] 重构后的记录逻辑能够正确计算应用的运行时间，包括持续运行的应用和跨天的情况
- [ ] 重构后的记录逻辑能够正确处理媒体应用的时间记录，包括不同音乐和视频的分开记录
- [ ] 重构后的记录逻辑能够正确处理电池状态等额外信息的更新
- [ ] 重构后的记录逻辑能够保持向后兼容性，不影响现有的其他功能
- [ ] 显示格式符合要求，美观、清晰、易读
- [ ] 持续运行的应用显示为 "now  起始时间（xx:xx）-现在 应用名称"
- [ ] 已结束的应用显示为 "起始时间（xx:xx）-结束时间（yy:yy） 应用名称"
- [ ] 跨天的应用显示格式正确，第一天结束的条目显示为 "起始时间（xx:xx）-次日 应用名称"，第二天新建的条目显示为 "now  昨日-现在 应用名称"
- [ ] 当天条目的运行时间和该应用在今天的总运行时间正确显示在条目末尾
- [ ] 音乐应用显示到媒体使用栏，不同音乐分开记录，同一音乐合并时间
- [ ] 视频应用在前台运行时显示到媒体使用栏并记录时间，不在前台运行时显示到媒体使用栏但不累计时间，不同视频分开记录，同一视频合并时间
